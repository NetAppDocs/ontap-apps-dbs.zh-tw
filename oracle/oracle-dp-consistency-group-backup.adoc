= 一致性群組備份
:allow-uri-read: 


[role="lead"]
為了實現最簡單的備份，只需將整個 Oracle 資料庫放在單一 ONTAP 磁碟區、LUN 或 NVMe 命名空間中即可。

使用單一快照保護資料檔案、歸檔日誌、重做日誌和控制檔案是一種有效的備份、還原和複寫方法。但是，RPO 僅限於備份本身的時間點。它適用於 RPO 為一小時或更長的情況。如果資料庫跨越 AFF 磁碟區或 ASA 儲存單元，則需要使用前面討論過的工具之一來建立一致性群組。

例如，整個資料庫可以位於單一 AFF 磁碟區中，並採用以下快照排程：

* 72 小時快照
* 30 個夜間快照
* 12 個月快照


這可以確保在過去 72 小時的滾動週期內，復原點目標（RPO）為 1 小時，此外還提供額外的夜間和每月備份。多個資料庫或應用程式檔案也可以包含在單一磁碟區或一致性群組快照集中，從而在更大的環境中實現一致的備份。
